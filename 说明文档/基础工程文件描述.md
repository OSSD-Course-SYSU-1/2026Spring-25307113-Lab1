## WindowOrientation 基础工程文件描述文档


## 项目概述
项目名称: WindowOrientation
包名: com.example.windoworientation
版本: 1.0.0 (versionCode: 1000000)
技术栈: HarmonyOS 6.0.2(22) + ArkUI + ArkTS
项目类型: 窗口方向适配示例应用

核心功能: 展示不同设备形态下的页面旋转策略，支持直板机、双折叠、三折叠、平板等多种设备形态的窗口方向适配。

## 技术架构总结
6.1 模块化架构
架构特点:

三层架构: commons(公共层) → features(功能层) → products(产品层)
HAR/HAP分离: commons和features打包为HAR，products打包为HAP
依赖管理: 通过oh-package.json5管理模块依赖
优势:

代码复用性高
模块独立开发测试
按需加载，减少包体积
6.2 状态管理
状态管理方案:

@State: 组件内部状态
@StorageLink: 全局状态共享(断点信息、播放器状态)
@Link: 父子组件双向绑定
AppStorage: 全局状态存储
应用场景:

窗口断点信息全局共享
播放器状态全局管理
折叠屏折痕区域信息共享
6.3 响应式布局
断点系统:

宽度断点: XS(超小屏) / SM(小屏) / MD(中屏) / LG(大屏) / XL(超大屏)
高度断点: SM(小屏) / LG(大屏)
断点范围: [320, 600, 840] vp
实现方式:

通过 UIContext.getWindowWidthBreakpoint() 获取当前断点
使用 BreakpointType<T> 根据断点返回不同值
使用 @StorageLink 监听断点变化
6.4 窗口方向适配
适配策略:

FOLLOW_DESKTOP: 跟随桌面方向(首页、股票)
PORTRAIT: 固定竖屏(竖屏游戏)
LANDSCAPE: 固定横屏(横屏游戏)
AUTO_ROTATION_UNSPECIFIED: 自动旋转(图库)
AUTO_ROTATION_LANDSCAPE_RESTRICTED: 自动旋转限制横屏(视频)
实现机制:

在 CardListViewModel 中为每个功能模块配置窗口方向策略
通过系统API设置窗口方向
